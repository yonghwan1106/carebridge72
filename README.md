# 케어브릿지 72 (CareBridge 72)

> **돌봄 공백을 72시간 안에 잇는 AI 의사결정 지원 서비스**
> 최중증 발달장애인 보호자의 입원·경조사·소진으로 돌봄이 끊긴 순간, 7종 공공데이터와 AI로 지자체 담당자에게 **0–2h · 2–24h · 24–72h 긴급 대응안**을 생성합니다.

**2026 사회보장정보와 공공·민간 빅데이터를 활용한 국민행복 서비스 발굴·창업경진대회** 출품작 작동 프로토타입
주최 보건복지부 / 주관 한국사회보장정보원·동국대학교

---

## ✨ 핵심 아이디어

기존 복지 서비스가 *"무엇을 받을 수 있는지"*를 안내했다면, 케어브릿지 72는 **"위기 상황에서 무엇을 어떤 순서로 연결해야 하는지"**를 제시합니다. 정보 안내가 아니라 **실행 대응안 생성**입니다.

서비스의 1차 사용자는 **지자체 통합돌봄·장애인복지 담당자**입니다. AI는 자동 배정·수급 결정을 하지 않고, 담당자가 검토·전화 확인·연계하도록 **추천과 설명**만 제공합니다(사람 중심 AI).

## 🧭 6단계 흐름

| 단계 | 내용 |
|---|---|
| ① 접수 | 담당자가 보호자 공백 상황을 자연어로 입력 |
| ② AI 상담 구조화 | 대상자·공백사유·예상시간·위험요인 추출 + 누락 질문 생성 |
| ③ 긴급도 분류 | 규칙 기반 안전장치 + AI로 즉시 / 당일 / 72h / 일반 판정 |
| ④ 72시간 대응안 | 0–2h·2–24h·24–72h 단계별 제도·기관 후보 + 담당자 확인 질문 |
| ⑤ 처리 기록 | 전화 확인 결과(연계/거절/대기)를 구조화해 축적 |
| ⑥ 사후 리포트 | 공급 부족 시간대·권역, KPI를 지자체 정책자료로 |

## 🤖 하이브리드 AI 아키텍처

```
[담당자 입력]
     │
     ▼
 ┌─────────────────────────────┐
 │  Claude (ANTHROPIC_API_KEY)  │  ← 비정형 상담 → 구조화 + 긴급도 분류 (NLU)
 │  ↳ 실패/미설정 시 규칙 엔진     │
 └─────────────────────────────┘
     │  structured + urgency
     ▼
 ┌─────────────────────────────┐
 │  공공데이터 매칭 엔진 (결정론적)  │  ← 72h 대응안·기관 후보·우선순위 점수
 │  RAG 그라운딩(실재 기관만 선택)  │
 └─────────────────────────────┘
     │
     ▼
 [72시간 대응안 + 후보 + 확인질문 + 사후 리포트]
```

- **하이브리드**: `ANTHROPIC_API_KEY`가 있으면 **실시간 Claude**로 상담을 구조화하고, 없거나 실패하면 **데이터셋 기반 규칙 엔진**으로 끊김 없이 작동합니다(키 없이도 100% 데모 가능).
- **결정론적 매칭**: 기관 우선순위는 *적합성·야간 가능·거리·품질평가·접근성*을 합산한 0–100점으로 산정합니다. AI가 기관을 자동 배정하지 않으므로 **환각·오배정 위험이 없고** 윤리 원칙과 일치합니다.

## 🚀 빠른 시작

```bash
# 1) 의존성 설치
npm install

# 2) 환경변수 (선택 — 실시간 Claude 사용 시)
cp .env.example .env.local
#   .env.local 의 ANTHROPIC_API_KEY 에 키 입력
#   키가 없어도 폴백 엔진으로 데모가 작동합니다.

# 3) 개발 서버
npm run dev
#   → http://localhost:3000

# (프로덕션)
npm run build && npm run start
```

| 환경변수 | 설명 | 기본값 |
|---|---|---|
| `ANTHROPIC_API_KEY` | Anthropic API 키 (없으면 폴백) | (없음) |
| `ANTHROPIC_MODEL` | 사용 모델 | `claude-haiku-4-5-20251001` (속도) / `claude-sonnet-4-6` (품질) |

> 스모크 테스트: 서버 실행 후 `node smoke-test.mjs` (성남시 야간 사례로 `/api/analyze` 호출·검증)

## ⚖️ 사람 중심 AI 원칙

AI는 추천·설명만 제공하고 **기관 배정·수급 결정은 담당자**가 합니다. 최소정보·동의 기반 처리, 실시간 수용 여부는 담당자 전화 확인 전제, 사후 분석은 비식별·집계 통계만 활용합니다.
